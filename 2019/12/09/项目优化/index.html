# 记一次项目优化经历

项目优化，可谓仁者见仁智者见智，不同的项目应根据项目的特点有针对性的优化，本篇以 vue spa 为例记录优化过程,以飨读者。
先说优化背景，最近在做一个实时视频的功能。用的腾讯的实时视频 TRTC sdk 及 IM sdk，由于是安卓端和 PC 端视频，安卓端需求多，web 端需求少，所以web端一周开发完毕，有足够多的时间去优化。期间也踩了不少坑，还算顺利。需要提醒下各位朋友的是，如果做 web 端实时视频，网站一定要用 https 证书，因为web端视屏通话用了最新的浏览器 API，navigator.mediaDevices.enumerateDevices() ，此 API 在 http 环境下会报错，https 下就可以正常使用。开发期间出现了，本地 demo 可以跑通，丢到服务器访问就报错的问题。官方文档也没有明确说明，耽误了不少时间。
web端开发完成了，暂时没事干就开始优化(摸鱼)，优化是参照 webpack 插件webpack-bundle-analyzer 打包结果分析的关于webpack打包分析可以看我这篇博客 。启动依赖图一看由于新引入了 trtc-js-sdk、cos-js-sdk-v5 和 tim-js-sdk 三个依赖，项目最大的 js 包压缩后大小都超过 1M 了,孰能忍。由于项目有登录页，就考虑在页面加载完成后就加载相关依赖，来提高用户体验，减少等待时间。到官网下载好js,准备开始。

1、webpack 配置依赖外置

  externals: {
  'vue': 'Vue',
  'element-ui': 'ELEMENT',
  'xlsx': 'XLSX',
  'trtc-js-sdk': 'TRTC',
  'tim-js-sdk': 'TIM',
  'cos-js-sdk-v5': 'COS'
  }

2、在 webpack 目录下 cdn.js 中添加依赖，如下参考

module.exports = {
  dev: {
    css: ['https://cdn.bootcss.com/element-ui/2.12.0/theme-chalk/index.css'],
    js: []
  },
  prod: {
    css: ['https://cdn.bootcss.com/element-ui/2.12.0/theme-chalk/index.css'],
    js: [
      'https://cdn.bootcss.com/vue/2.6.10/vue.min.js',
      'https://cdn.bootcss.com/element-ui/2.12.0/index.js',
      '/static/js/load-js-v1.0.0.js'
    ]
  }
}

视频相关三个依赖统一放到 load-js 里加载,load-js 内容如下：

;(function(win, doc) {
  win.addEventListener(
    'load',
    function() {
      var timSDK = '/static/js/tim-js-v2.2.1.js'
      var trtcSDK = '/static/js/trtc-v4.3.1.js'
      var cosSDK = '/static/js/cos-js-sdk-v5-v0.5.22.js'

      addScript(timSDK)
      addScript(cosSDK)
      addScript(trtcSDK)
    },
    false
  )

  function addScript(url) {
    var script = doc.createElement('script')
    script.src = url
    doc.body.appendChild(script)
    return script
  }
})(window, document)

3、在webpack中使用

const cdn = require('./cdn')
new HtmlWebpackPlugin({
  ...
  cdn: cdn.prod
})

4、在html注入依赖 因为 html-webpack-plugin 支持lodsah模板字符串，所以可以在html模板中使用 lodash template 语法，参考如下：

<head>
  <% if(htmlWebpackPlugin.options.cdn) {%> 
    <% for(let css of htmlWebpackPlugin.options.cdn.css) { %>
      <link rel="stylesheet" href="<%=css%>" />
    <% } %>
  <% } %>
</head>
<body>
  <% if(htmlWebpackPlugin.options.cdn) {%>
      <% for(let js of htmlWebpackPlugin.options.cdn.js) { %>
        <script src="<%=js%>"></script>
      <% } %>
  <% } %>
</body>

注：在步骤 2 中，公司有 cdn 就用公司 cdn，没有的话就放服务下访问好，免费的 cdn 不敢用，一旦出问题这个锅背不起

在这个过程中遇到一点小问题，在页面进入系统，没问题，很完美，但是刷新页面后 js 就报错,tim 依赖找不到，原因定位是tim.js 依赖还没有加载就调用 tim.js，因为使用了 window.onload 事件。然后想到 import() 懒加载相关依赖。但是怎么知道 trtc 依赖已经加载完，最后通过 new Event 自定义事件完美解决(有兼容性问题，但实时视频要求使用 chrome浏览器 且版本不低于 72 所以这个问题可以忽略)，load-js最终代码如下：

;(function(win, doc) {
  win.addEventListener(
    'load',
    function() {
      var timSDK = '/static/js/tim-js-v2.2.1.js'
      var trtcSDK = '/static/js/trtc-v4.3.1.js'
      var cosSDK = '/static/js/cos-js-sdk-v5-v0.5.22.js'

      if (typeof Event == 'object') { // 跳过IE浏览器
        return
      }

      var myUtilEvent = new Event('jsLoadComplete') // 注册自定义事件
      var root = doc.body
      var tim = addScript(timSDK)
      var cos = addScript(cosSDK)
      var trtc = addScript(trtcSDK)

      trtc.onload = function(e) {
        root.dispatchEvent(myUtilEvent) // 在body上触发自定义事件
      }
    },
    false
  )

  function addScript(url) {
    var script = doc.createElement('script')
    script.src = url
    doc.body.appendChild(script)
    return script
  }
})(window, document)

由于刷新和正常加载走的不同的逻辑，需要一个存 sessionStorage 作为区分标志，采取不同的加载策略,vue 代码如下:

// 在created生命周期中
var isReload = sessionStorage.getItem(sessionKey.isReload) // 区分是否刷新存一个sessionStorage 作为判断依据
if (!isReload) {
  this.initVedio()
  sessionStorage.setItem(sessionKey.isReload, true)
} else {
  this.$nextTick(function () {
    let root = document.body
    root.addEventListener('jsLoadComplete', this.initVedio, false) // 监听body自定义事件
  })
}

// methods 方法
initVedio () {
  import('@/view/consultDoctor/im').then(obj => {
    ...
      obj.createIM() // 在回调里调用依赖的方法
    ...  
  })
  let isReload = sessionStorage.getItem(sessionKey.isReload) 
  if (isReload) {
    let root = document.body
    root.removeEventListener('jsLoadComplete', this.initVedio) // 触发后移除自定义事件
  }
}

通过以上一顿骚操作之后，再打包看依赖图，最大的js文件压缩后只有350kb左右,开启gizp压缩后只有100kb,对于PC端来说是一个可以接受的范围。说明优化见效果了。

总结：项目优化，有很多方向，比如首屏优化，本案例也涉到，例如等页面加载完成，再加载相关依赖。缓存优化，依赖按需加载，依赖懒加载，图片懒加载等等。具体的项目可以根据实际的需求灵活选择。需要说的是项目优化要以实际项目运行数据为基准，任何过度的优化都可能适得其反。最终以项目资源加载速度，耗时，以及用户体验的流畅度作为评判标准。望每个前端工程师都能写出用户体验极棒的项目。各位大神若有高见欢迎后台留言交流切磋。以上，谢谢！
